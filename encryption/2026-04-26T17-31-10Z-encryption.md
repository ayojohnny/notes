# Encryption

plaintext: what is being encrypted (i.e. unecrypted data)
ciphertext: the resulting encrypted text

Simple encryption:
- uses the same encryption algorithms to convert to plaintext into cipher text
- if you know the algorithm, you can steal any information

plaintext -> apply encryption algo  -> cipher text (bob)
                                    -> cipher text (alice)

Key based algorithms:
- same algorithm used to encrypt
- a key is randomly generated

Despite the same secret being used (let's say a password - foobar), each receiver
has a different cipher text. This is because the underlying encryption algorithm is
being used in conjunction with random generated key. This results in 3 different
cipher texts as shown below.

plaintext -> apply encryption algo + random generated key   -> js12jsw8 (bob)
          -> apply encryption algo + random generated key   -> hg8siPds (alice)
          -> apply encryption algo + random generated key   -> xuqDia3z (james)
